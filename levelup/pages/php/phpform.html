Building a simple form using PHP and MySQL

Part 1

February 11, 2020 Lazar RistiÄ‡

In this article, I will be showing you how to do some basic stuff with forms in HTML, using PHP and MySQL.

I learned this from Tania Rascia's blog, so a lot of things will be similar, only here I will also tackle working with images and login/registration.

You will need some basic knowledge of HTML and CSS to keep up with this article.


1. Setting up local server and phpMyAdmin


Before we even get started with coding, I will need you to download WampServer so you can create your own server locally. When you install it, you just need to run it and it will automatically set up everything for you.

Next up, in your browser type localhost/phpmyadmin and it will take you to the login page of phpMyAdmin. Username is root, there is no password. When you log in, you will see a vertical tab on the left with all the databases and at the top it will say New. In the input field that says Database name type realestate, since the database and form will be real-estate oriented. Click Create and after that when the new screen loads, create a table called houses that has 4 columns. Three of these columns will be for the location, price and size, and the fourth one will be an ID of a house.

On the new screen, fill the first row like this:
Name -> id, Type-> INT, Length/Values -> 10 and check an A_I field (A_I means Auto_Increment which will automatically increase the id with every house entered in the database). You will get a pop up after you check A_I, click Go
Fill the second row:
Name -> location, Type->VARCHAR, Length/Values -> 20
Fill the third row:
Name -> price, Type->VARCHAR, Length/Values -> 20
Fill the fourth row:
Name -> size, Type->VARCHAR, Length/Values -> 20

After this, click Save and the table houses will be created.

Now, on the left tab, click on houses and then at the top of the screen you will find a tab called Insert, click on it. Here in the Value fields we will enter 1, New York, 500, 60 as id, location, price, size, respectively. After that, click Go and we're ready to go. :)

The method="post" property tells the server how will the data from the form be sent. We have two ways of sending data, with POST or with GET.

With POST you don't see the data being sent in the url, while with GET you can see the data that is being sent. Normally, using GET isn't good when we are working with sensitive, private data. In some tutorials you may see action="something" in the form tag and that is used when you want to show results on a different webpage. Here we won't be using action property, since we will show the results on the same page.

The input tag: input tag always consists of 3 attributes; type, name and id. Type depends on the type of options we are giving to the user; type="text" lets the user input some text, type="radio" creates a radio button, type="checkbox" creates a checkbox, and so on. You can change the type of input depending on your needs.

We are using 3 text inputs and one submit input. The submit input will send the data from the form to the server. It's value basically says what will be written inside the button.

Next up, adding MySQL.

So, our data has been submitted in the form and now we want to put it in a query and send it to a database that we connected to with our $connection variable. Underneath it, we put an MySQL query: $sql = "SELECT * FROM houses WHERE location = :location AND price = :price AND size = :size;"

These :location, :price and :size represent parameters that will be binded, so you can look at it as '?' since it will change depending on the data that will be sent from the form.

Next, we store the data sent from the form into the variables. So, we will have
$location = $_POST['location'];
$price = $_POST['price'];
$size = $_POST['size'];
Location, price and size held in $_POST[''] are actually names of inputs in the form. So if we had, for example, an input with a name roomNumber, we would put $roomNumber = $_POST['roomNumber'].

Now that we've retrieved data from the form, we can bind our parameters in the SQL query. When we have parameter binding, we use prepared statements.

First, we will prepare our SQL query: $statement = $connection->prepare($sql);

Then, we will bind our parameters, one by one:
$statement->bindParam(":location", $location, PDO::PARAM_STR);
$statement->bindParam(":price", $price, PDO::PARAM_STR);
$statement->bindParam(":size", $size, PDO::PARAM_STR);
Basically, what bindParam() function does is, it assigns the value from the variable to the parameter in the prepared statement. The third attribute, PDO::PARAM_STR says just that, bind the PARAMeter to the STRing.

After binding, all we have to do is execute the statement:
$statement->execute();

Now that the statement has been executed, we will fetch all the results:
$result = $statement->fetchAll();

With this, we finish our PHP code above the HTML for now and go to HTML where we will embed some more PHP into the HTML.

What this does is, if we submitted a form, it checks if result fetched anything at all and if number of rows fetched is bigger than 0.
Basically, you can only check $statement->rowCount() > 0.

Afterwards, if it worked, we print everything that is returned. Since $result is an array, we go into foreach loop that goes through that array and then we echo each column from the table houses. Here I didn't echo id column, but you can do that too.

And before we finally press that Submit button, we just need to write a config.php file and put an require "config.php" above the try-catch block.
